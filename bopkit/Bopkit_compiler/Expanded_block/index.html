Module Bopkit_compiler.Expanded_block

This is an internal representation used as an intermediate step in the chain of transformations that leads to the final cds.

An Expanded_block.t is quite close to the blocks found in an Expanded_netlist.t, except we've distinguished the primitives from other calls, and we've done more context verifications, such as arity checks.

The environment formed by all Expanded_block.t indexed by their name is then used to create the Expanded_nodes.t structure.

type call =
  1. | Block of {
    1. name : string;
    }
  2. | Primitive of {
    1. gate_kind : Bopkit_circuit.Gate_kind.t;
    }
val sexp_of_call : call -> Sexplib0.Sexp.t
type node = {
  1. call : call;
  2. inputs : string list;
  3. outputs : string list;
}
val sexp_of_node : node -> Sexplib0.Sexp.t
type t = {
  1. loc : Loc.t;
  2. name : string;
  3. local_variables : string list;
  4. input_names : string array;
  5. output_names : string array;
  6. nodes : node list;
}
val sexp_of_t : t -> Sexplib0.Sexp.t
type env = t Core.Map.M(Core.String).t

An environment of expanded_blocks, indexed by their name. This allows for performing the lookups when encountering a call equal Block { name } in the body of another block.
