Module Bopkit.Expanded_netlist

Expanded netlist is an intermediate representation built during the analysis of a circuit. Constructs such as control structures and imbrications have been developed. This representation is used by different parts of bopkit such as the simulator, bop2c, etc.

type memory = {
  1. loc : Loc.t;
  2. name : Base.string;
  3. memory_kind : Netlist.memory_kind;
  4. address_width : Base.int;
  5. data_width : Base.int;
  6. memory_content : Bit_matrix.t Base.option;
}
type external_method = {
  1. method_name : Base.string;
  2. attributes : Base.string Base.list;
}
type external_block = {
  1. loc : Loc.t;
  2. name : Base.string;
  3. attributes : Base.string Base.list;
  4. init_messages : Base.string Base.list;
  5. methods : external_method Base.list;
  6. command : Base.string;
}

Once the control structures have been developed, all the indexes are computed and are integers.

type index =
  1. | Interval of Base.int * Base.int
  2. | Index of Base.int
val equal_index : index -> index -> Base.bool
val sexp_of_index : index -> Sexplib0.Sexp.t
type variable =
  1. | Signal of {
    1. name : Base.string;
    }
  2. | Bus of {
    1. loc : Loc.t;
    2. name : Base.string;
    3. indexes : index Base.list;
    }
  3. | Internal of Base.int
type variables = {
  1. expanded : Base.string Base.list;
  2. original_grouping : variable Base.list;
}
type call =
  1. | Block of {
    1. name : Base.string;
    }
  2. | External_block of {
    1. name : Base.string;
    2. method_name : Base.string Base.option;
    3. external_arguments : Base.string Base.list;
    }
type node = {
  1. loc : Loc.t;
  2. call : call;
  3. inputs : variables;
  4. outputs : variables;
}

Flat nodes where the nested inputs have all been developed.

type block = {
  1. loc : Loc.t;
  2. name : Base.string;
  3. attributes : Base.string Base.list;
  4. inputs : variables;
  5. outputs : variables;
  6. unused_variables : variables;
  7. nodes : node Base.list;
}

Blocks with parameters are produced on demand, with a fresh name for each variation of required parameters. At the moment, the actual name is made of a concatenation of the original name plus additional characters taken from the concrete syntax, such as '', '', etc to make it unique. Example: "name4".

type t = {

  3. memories : memory Base.array;
  4. external_blocks : external_block Base.list;

}
